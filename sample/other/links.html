External Elf Support:


An external elf can be linked to a particular html page using the html link tag:
<link type="binary/elf" href="FILE_NAME.elf">

Be aware that:

  • Only the first linked elf file will be loaded, additional links are ignored.
  • Any href argument value found ending with ".elf" in an anchor or div tag will trigger the elf function handling, and this value does not necessarily need to match the elf file name linked to the page.

Single Run Functions:

To use a static external function which is to run once when clicked, use the html tag:
<link type="binary/elf" href="FILE_NAME.elf">
then provide the anchor:
<a href="FILE_NAME.elf" type="function/static" target="FUNCTION_NAME_FROM_ELF">


Try the following examples.

  • func1
  • func2
  • func3
  • func4

Repeatedly Run Functions:

To use a dynamic external function which is to run repeatedly once clicked, use the html tag:
<link type="binary/elf" href="FILE_NAME.elf">
then provide the anchor:
<a href="FILE_NAME.elf" type="function/dynamic" target="FUNCTION_NAME_FROM_ELF">

A dynamic function must return an int and return 0 when it should stop, repeated starts are ignored.


Try the following examples.

  • func_d1
  • func_d2

Always Run Functions:

To use an always running external function, use the html tag:
<link type="binary/elf" href="FILE_NAME.elf">
then provide the text tag:
<text href="FILE_NAME.elf" type="function/always_run" target="FUNCTION_NAME_FROM_ELF">

An always running function is started on page loading to run continuously and may be configured to return a string which is to be displayed in the html text tag; to enable this provide a rel="string" argument in the text tag and write your function of the form:
void FUNCTION_NAME_FROM_ELF(char *buf, int size)


Example:

func_always
